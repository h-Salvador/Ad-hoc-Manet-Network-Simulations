Performance Metrics

A. Throughput

Throughput measures the rate at which data can be transmitted over a network.
It represents the amount of data that can be successfully delivered from the source to the destination per unit of time.
Throughput is typically measured in megabits per second (Mbps). A higher throughput indicates better network performance, as it indicates a network's capacity to handle a larger volume of data.

B. Packet Delivery Ratio

The Packet Delivery Ratio (PDR) is the ratio of the number of data packets successfully delivered to the destination nodes to the total number of data packets sent by the source nodes.
It represents the effectiveness of data delivery in the network.
A higher PDR indicates better performance, as it implies a higher percentage of successfully delivered packets.

C. End to End Delay

End-to-End Delay is the average time interval between the generation of packets in a source node and their successful delivery to the destination node.
It measures the time it takes for a packet to traverse the network from the source to the destination.
A lower end-to-end delay indicates better performance, as it reflects faster data transmission and reduced latency.

D. Number of Packets Dropped

The Number of Packets Dropped refers to the count of data packets that are not successfully delivered to the destination during transmission.
Packet drops can occur due to various reasons, such as network congestion, errors, or limitations in the network infrastructure.
Minimizing the number of dropped packets is essential for maintaining high network performance and data integrity.

E. Jitter

Jitter is a measure of the variation in packet delay between different nodes in the network. It quantifies the difference in arrival times of packets at the destination.
A higher jitter value indicates more significant variations in packet delay, which can lead to inconsistent and unpredictable network performance.
Minimizing jitter is crucial for achieving stable and reliable data transmission.
